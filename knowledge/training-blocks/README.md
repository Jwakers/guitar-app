# Training Blocks

This directory contains one document per training block type.

Training blocks are multi-week structured programmes. Each block type represents a distinct training philosophy and weekly structure. The adaptive engine uses block documents as the basis for generating weekly plans and session prescriptions.

**Practice methodology:** Block design must align with [`knowledge/principles/practice-methodology.md`](../principles/practice-methodology.md) — interleaved sessions, chunked duration presets, edge-of-ability targeting, and troubleshooting-oriented light/deload sessions.

---

## Practice Design Constraints

Training blocks must respect evidence-informed practice design:

- **Interleaving** — weekly sessions should rotate complementary skills, not prescribe one technical idea for an entire session unless explicitly a repair micro-session.
- **Session chunking** — block templates should define how 15–45 minute sessions divide into warm-up, primary focus, interleaved support, benchmark, and reflection sections.
- **Edge-of-ability** — block progression should calibrate targets from Reliable Performance + small steps, not peak scores.
- **Deload / light sessions** — align with troubleshooting and control work when recent logs show repeated failure or fatigue.

## Document Template

Each training block document must define the following sections:

```markdown
# [Block Name]

## Block Objective
The primary training goal of this block type.

## Typical Duration
Recommended number of weeks.

## Weekly Structure
The pattern of session types across a typical week
(e.g. standard / standard / light / standard / test).

## Target Workload
Low | Moderate | High — overall training load for this block.

## Primary Skills
The skill areas this block develops most intensively.

## Secondary Skills
Supporting skills trained at lower volume.

## Session Chunking
How a typical session within this block is divided
(e.g. warm-up / primary focus / interleaved support / benchmark / reflection).

## Interleaving Strategy
Which complementary skills rotate within a session for this block.

## Edge-of-Ability Targeting
How this block calibrates difficulty relative to Reliable Performance.

## Progression Philosophy
How the block increases difficulty across its duration.

## Deload Strategy
When and how a deload week is introduced within this block.

## Completion Criteria
The conditions that indicate a player has completed this block successfully.

## Typical Next Block
The recommended block type to follow this one.
```
